## 一、四条技术路径对比

### KAIROS (S&P 2024)

| 步骤 | 做法 |
|------|------|
| 1. 输入 | 流式 OS 审计日志 |
| 2. 图处理 | 事件作为动态时序图持续处理，更新边和节点的记忆状态 |
| 3. 特征 | 节点记忆嵌入 + 原始边特征 (39维: 16维src hash + 7维边one-hot + 16维dst hash) |
| 4. 模型 | TGN (TransformerConv + 时序记忆模块) 在良性数据上训练，预测边类型 |
| 5. 检测 | 重建 loss 高的边被分组到固定 15 分钟窗口，用 mean+1.5σ 统计阈值 + IDF + 关键词黑名单过滤 |
| 6. 输出 | 15 分钟时间窗口 + 窗口内异常边子图 |

**核心假设**: "模型学会正常事件过渡 → 偏离的事件是异常 → 按时间聚合找攻击"

### MAGIC (USENIX Security 2024)

| 步骤 | 做法 |
|------|------|
| 1. 输入 | OS 审计日志 |
| 2. 图处理 | 时序日志被压平为静态图快照，丢弃执行时间间隔 |
| 3. 特征 | 节点类型和边类型的简单 one-hot 编码 |
| 4. 模型 | 图掩码自编码器 (GMAE = GAT encoder + decoder)，在良性快照上训练，重建被 mask 的节点特征 |
| 5. 检测 | KNN 密度估计 — latent space 中离良性 cluster 太远的节点被标记 |
| 6. 输出 | 可疑实体列表（节点级） |

**核心假设**: "学习正常实体表示 → 偏离正常分布的实体是异常"

### EagleEye (eCrime 2024)

| 步骤 | 做法 |
|------|------|
| 1. 输入 | OS 审计日志 |
| 2. 图处理 | 事件被线性化并切成固定大小序列窗口，忽略图拓扑结构 |
| 3. 特征 | ~60 维手工安全特征 per event（含命令行语义嵌入） |
| 4. 模型 | Encoder-only Transformer (BERT-tiny)，在混合良/恶性窗口上监督训练 |
| 5. 检测 | [CLS] token 二分类头输出恶意/良性分数 |
| 6. 输出 | 被标记为恶意的序列窗口 |

**核心假设**: "系统日志像自然语言 → Transformer 自注意力能捕获长距离依赖"

### 我们的方法：因果边感知时序链提取

| 步骤 | 做法 |
|------|------|
| 1. 输入 | 流式 OS 审计日志 |
| 2. 图处理 | **从 TGN 标记的种子边出发，沿有向因果边双向追溯，用分支加权因果相干性度量 (Φ_bw) 剪枝** |
| 3. 特征 | **75 维向量：层次路径 hash + 正弦时间编码 + 图距离嵌入 + 桥接类型标志 + 结构标记** |
| 4. 模型 | **自回归 Transformer，仅在良性链上自监督训练，多任务预测：下一个事件的类型 + 节点特征 + 时间间隔** |
| 5. 检测 | **分量级 eCDF 阈值 — 对自回归预测损失的每个分量 (语义/结构/时间) 独立校准，取 max** |
| 6. 输出 | **精确的、不间断的、多步因果攻击事件链** |

**核心假设**: "攻击是因果链 → 定义什么是因果上好的链 → 按定义提取 → 用模型验证"

## 二、路径分叉分析

### 三个 baseline 在哪里汇合？

**它们都在 Step 2 汇合** — 无论是切成时间窗口 (KAIROS)、压成静态快照 (MAGIC)、还是截成固定序列 (EagleEye)，它们**全部切断了多步攻击的真正因果依赖**。

### 我们在哪里分叉？

- **对 KAIROS**: 分叉在 Step 2（不按时间桶，按因果追溯）和 Step 5（不用 1.5σ，用学习的自回归 loss）
- **对 MAGIC**: 分叉在 Step 2（拒绝静态快照，保留精确时序）
- **对 EagleEye**: 分叉在 Step 2（拒绝盲切窗口，沿图拓扑追溯）和 Step 4（拒绝监督分类，用自监督序列预测）

### 创新集中在哪里？

**集中在 Step 2**。这不是 KAIROS 的 TGN + EagleEye 的 Transformer 拼在一起。如果直接拼接，高分支 OS 进程（编译器、浏览器）会产生数百万条"因果相连"的事件，塞爆 Transformer 的上下文窗口。**我们的根本突破是模型无关的 Φ_bw 校准** — 在 ML 模型看到数据之前就解决了组合爆炸问题。这是非增量性的根源。

## 三、10 分钟中文汇报讲稿

### (0:00 - 1:00) 开场

"请大家看一个真实的 DARPA 攻击。一个 Nginx 漏洞被利用，fork 出一个 bash shell。这个 shell 执行了恶意脚本，下载了 payload。Payload 读取了 `/etc/passwd`。最后，把数据发送到了外部 IP `81.49.200.166`。

六步。每一步导致下一步。

这是攻击真实发生的方式。

但现在，我想给大家看看——三个最先进的检测系统，在看这同一个攻击时，**实际上看到了什么**。"

### (1:00 - 3:00) 三条路径

"KAIROS——去年 S&P 的论文——它看到了什么？它看到这六步攻击是一堆统计数据，塞在一个 15 分钟的时间窗口里。这个窗口里同时还有浏览器刷新、系统日志轮转、定时任务执行——总共上万个无关事件。攻击被稀释了。**KAIROS 把因果链换成了时间桶。**

MAGIC——USENIX Security 的论文——它看到了什么？它在溯源图上成功标记了 `/tmp/vUgefal` 这个恶意文件。但它丢弃了时间轴。它不知道这个文件是谁 spawn 出来的，也不知道这个文件接下来读了什么、发给了谁。分析员拿到的是一个**嫌疑人名单**，不是**作案过程**。**MAGIC 把因果链换成了实体列表。**

EagleEye——它看到了什么？它把系统日志切成固定大小的窗口。因为这次攻击正好跨了两个窗口的边界——前一半在窗口 A，后一半在窗口 B。因果连接被算法本身切断了。**EagleEye 把因果链换成了文本片段。**

三家顶会。三条路径。一个共同的代价：**它们都把 provenance graph 最核心的价值——因果边——给丢掉了。**"

### (3:00 - 5:00) 我们的分叉

"我们的方法，在管线的第二步——图处理这一步——和它们彻底分开了。

我们不做时间切片。不做静态快照。不做固定窗口。

**我们追溯因果边。**

当 Nginx fork 出 bash，内核记录了一条 FORK 边。我们跟着它走。当 bash exec 恶意程序，内核记录了一条 EXEC 边。我们跟着它走。不是猜的——内核保证了因果。

但这里有一个致命问题：如果无脑追溯，一个 Firefox 进程开一个网页可能产生上万条因果相连的文件读写事件。所以我们设计了一个**因果相干性度量 Φ_bw**——它做三件事：

第一，区分桥接类型。进程桥接——内核保证因果关系，时间间隔不重要，不施加时间惩罚。文件桥接——共享资源，可能被第三方覆盖，时间越久越不可靠，施加时间衰减。

第二，惩罚分支。攻击链——nginx exploit → shell → wget → exfiltrate——分支因子约 2.5。Firefox 开网页——分支因子 15。我们用 exp(-λ·BF) 把良性高分支进程压下去，攻击链浮上来。

第三，参数自动校准。δ、β、λ 全部通过肘部检测在良性数据上自动发现。不需要人工调参。不需要攻击标签。

然后——第四步——我们用一个自回归 Transformer 验证提取出来的链。不是 1.5σ。不是 KNN 距离。Transformer 在良性链上学习预测事件过渡——下一个事件应该是什么类型？时间间隔应该是多少？如果一条链让模型持续做出非常规预测，它就是异常的。**检测是学习的，不是调的。**"

### (5:00 - 6:00) 为什么这条路径能工作四个理由。

第一，信号密度。我们的链保证连续事件之间 d ≤ δ。固定窗口里 10,000 个事件，大部分和攻击无关。我们的链里 50 个事件，每个都和攻击因果相连。

第二，因果不变性。我们的特征建在内核保证的进程转换上。KAIROS 和 EagleEye 的特征建在时间共现上——巧合。进程 fork 关系跨环境稳定，时间共现随系统负载变化。所以我们的特征应该迁移更好。这是一个可检验的预测。

第三，注意力互证。当链上第 5 个事件是 'SENDTO 外部 IP'，Transformer 的自注意力机制会放大第 3 个事件 'READ /etc/passwd' 的可疑度——因为它们是真正因果相连的。在固定窗口里，这种放大可能是虚假的——因为事件 3 和事件 5 可能毫无关系，只是恰好落在同一个窗口里。

第四，容量效率。O(N²) attention。10,000 个事件 → 计算爆炸。50 个事件 → Transformer 的全部表示能力聚焦在攻击轨迹上。"

### (6:00 - 6:30) 类比

"打个比方。追踪 APT 就像追踪一架敌机。

KAIROS 告诉你哪个 15 分钟时段有奇怪的雷达扰动。MAGIC 告诉你哪个跑道标识看起来不对劲。EagleEye 录下了天空中所有无线电通讯的原始记录，混杂了三架民航机的信号。

**我们重建了那架飞机的完整飞行路径。从起飞到降落。**"

### (6:30 - 7:30) 实验验证

"我们的实验直接对应这个架构的五层论证。

第一，Φ_bw 真的在度量因果结构——良性链 vs 随机链的 AUC，模型无关，训练之前就能验证。

第二，肘部检测真的发现了 OS 的自然因果视野——δ 不需要人设定，数据说了算。

第三，因果链在所有分割策略中胜出——链级 F1 对固定窗口、随机游走、时序子图。

第四，因果特征跨环境迁移——E3 训练，E5 测试。因为因果机制是稳定的，而时间共现不是。

第五，每个组件都在消融实验中证明了自己的价值。"

### (7:30 - 8:00) 收尾

"KAIROS、MAGIC、EagleEye 都推动了这个领域。但它们沿着三条平行的轨道前进——时序的、结构的、序列的——从未交汇。

我们的路径使它们交汇。交汇点是因果链。

我们不再问'哪里有异常'。我们开始回答'攻击是什么'。"

## 四、电梯演讲 (30 秒)

"当前 APT 检测系统把攻击事件塞进时间窗口、压成静态节点、或切成文本片段——全都丢掉了攻击的因果结构。我们直接从 OS 内核记录的因果边追溯攻击链，用一个模型无关的相干性度量过滤噪声，再让自监督 Transformer 验证整条链的异常性。结果：不需要攻击标签，输出完整的攻击叙事——谁干的、怎么干的、按什么顺序。"

## 五、核心论点一句话

> 三个 baseline 把因果链分别换成了时间桶、实体列表、文本片段。我们把因果链本身变成了检测单元。

## 六、关键防御 (应对质疑)

| 可能的攻击 | 防御 |
|-----------|------|
| "这不就是 KAIROS + EagleEye 拼一起？" | 如果直接拼接，Firefox 的高分支会让 Transformer 崩溃。Φ_bw 在模型看到数据之前就解决了组合爆炸。这是 Step 2 的创新，不是拼凑。 |
| "3-case φ 看起来很 ad-hoc" | 三个 case 映射到 OS 的不可变原理：进程是 capability，文件是 shared resource。不是拍脑袋，是 OS 教科书。参数由肘部检测自动发现，不存在人工调参。 |
| "只训练在良性数据上，复杂良性更新不会误报吗？" | Φ_bw 的分支惩罚 (Structural Sparsity Prior) 在提取阶段就把高分支良性行为压下去了——它们根本不会进入 Transformer。 |
| "DARPA E3 只有一个攻击场景" | 我们承认。用 E5 跨数据集 + 因果不变性预测（因果特征迁移更好）来缓解。OpTC 是未来的工作。 |
