% Introduction
\section{Introduction}
\label{sec:introduction}

Advanced Persistent Threats (APTs) typically unfold as multi-stage campaigns: an adversary first obtains an initial foothold, then performs persistence, privilege escalation, lateral movement, and data exfiltration~\cite{mitreatt}. Individual events in this process are often not rare. Launching a shell, reading a configuration file, opening a network connection, or writing to a temporary directory can all be legitimate operations on a production host. What separates an attack from benign activity is often not a single event, but whether a sequence of such events forms an interpretable causal chain.

System provenance graphs provide a natural data source for recovering such chains. Modern operating-system kernels mediate system calls and record dependencies among processes, files, and network objects. Organizing these audit records as a directed graph yields nodes for system entities and edges for kernel-mediated dependency events. Compared with application logs or aggregate statistics, provenance graphs have two important advantages: within an explicitly stated audit model, they nearly cover the system interactions mediated by the kernel; and unless the attacker already has kernel-level capability, the corresponding audit records are difficult to tamper with.

Directly following dependencies in a provenance graph, however, leads to \emph{dependency explosion}~\cite{kingbacktracking}: expanding forward or backward from a suspicious event can quickly include large amounts of unrelated benign activity. Existing provenance-based intrusion detection systems (PIDS) mostly detect anomalies at the graph, window, entity, or edge level~\cite{unicorn2020,yang2023prographer,cheng2024kairos,jia2024magic,wang2022threatrace,edgetrace2025}, leaving attack-chain reconstruction to downstream post-processing or manual investigation. Recent Transformer-based methods over provenance data typically construct sequences using fixed time windows, random walks, or temporal subgraphs~\cite{eagleeye2024,sentient2026,getaid2025}; these sequences need not respect provenance-edge direction or entity-identity continuity. Chain-level APT detection therefore still lacks an upstream definition: \emph{which event sequences should count as structurally valid causal chains?}

This paper answers that question. We first state an audit model in \S\ref{sec:audit_model}, covering the trusted computing base, edge semantics, entity identity, and missing-event handling. Under this model, we define a chain coherence score $\branchCoherence$ that requires neither attack labels nor a trained model. The score is not a maliciousness score and is not interpreted as a probability; it measures whether adjacent chain links are supported by observable dependency evidence in the audit log. We then build a chain-level detection pipeline: an edge-level seed scorer selects events worth investigating, $\branchCoherence$ guides bidirectional causal-chain extraction, and a self-supervised sequence model trained on benign chains produces the anomaly score $A(\chain)$.

Our contributions follow a three-layer structure---problem formulation, chain extraction, and end-to-end validation:
\begin{enumerate}
\item \textbf{We reformulate APT detection as causal-chain recovery.} Unlike window-, entity-, or edge-level alerting, our work directly treats the event chain needed for investigation as the detection object. We separate two questions: whether a chain is structurally valid on the provenance graph, and whether a structurally valid chain deviates from benign behavior. The former is bounded by kernel-audit semantics; the latter is learned from the benign distribution. This decomposition explains why chain-level detection is well-founded: both benign and attack chains must obey system dependency structure, but attack chains are semantically anomalous.
\item \textbf{We propose a method for extracting trustworthy causal chains from large provenance graphs.} To address dependency explosion, we do not blindly expand graph reachability. Instead, $\branchCoherence$ scores whether adjacent events have sufficient dependency support, and uses that score to rank, prune, and truncate candidate chains. The score depends only on observable audit-log evidence---including process identity continuity, whether shared resources are perturbed by third parties, and the finite identity scope of system entities---so it can be calibrated without attack labels or attack templates.
\item \textbf{We build and evaluate a zero-attack-prior chain-level APT detection system.} The system first extracts structurally credible candidate chains, then uses a self-supervised sequence model to detect semantic anomalies, and evaluates outputs with chain-level metrics for attack relevance and complete narrative recovery. On DARPA Transparent Computing data, we report end-to-end detection, ablation, cross-engagement generalization, and efficiency results, showing that chain-level outputs can reduce analyst investigation cost more directly than traditional window or node alerts.
\end{enumerate}
