\section{Related Work}
\label{sec:related}

We organize related work by the detection logic for APTs rather than by the specific model architecture. Existing work can be broadly grouped into two lines. One line starts from anomalous behavior: it learns normal system behavior and searches for deviations. The other starts from the attack process: it attempts to recover paths, sequences, or subgraphs that describe how an attack unfolds. Our work is connected to both lines. We use anomaly detection to identify suspicious starting points, but lift the final detection object to causal chains and apply chain-level modeling to decide whether the recovered process is anomalous.

\paragraph{Anomaly-driven APT detection.}

The first line treats APT detection as anomaly discovery: it learns representations of benign system behavior and detects deviations at the graph, time-window, node, entity, or edge level. UNICORN~\cite{unicorn2020} uses Weisfeiler--Lehman graph kernels and histogram sketching to classify entire provenance graphs. PROGRAPHER~\cite{yang2023prographer} partitions provenance into temporal snapshots and applies graph2vec with TextRCNN for sequence-level anomaly detection. MAGIC~\cite{jia2024magic} learns benign entity representations using a graph masked autoencoder and detects anomalous entities with KNN density estimation. ThreaTrace~\cite{wang2022threatrace} and FLASH~\cite{flash2024} perform node-level detection using GraphSAGE or semantic/contextual graph encoders. KAIROS~\cite{cheng2024kairos} computes per-edge reconstruction loss with a Temporal Graph Network and aggregates abnormal signals within time windows. EdgeTrace~\cite{edgetrace2025} also uses edge-level anomaly scoring as the basis for subsequent tracing.

The strength of this line is that it can locate ``where something is abnormal'' in large audit streams: anomalous time periods, suspicious entities, nodes, or edges. However, the detection object is usually a local or aggregated unit rather than the complete behavioral process needed for attack investigation. Analysts still need to determine after the fact whether these local alerts are causally connected. Our work inherits the detection capability of this line: in the current instantiation, we use KAIROS's edge-level score to provide seed events. The difference is that edge anomalies only indicate where investigation should start; final detection and evaluation are performed on event chains extracted under causal constraints.

\paragraph{Attack-process-driven detection and investigation.}

The second line directly targets the attack process by organizing scattered events into paths, sequences, or subgraphs. Holmes~\cite{holmes2019}, Poirot~\cite{poirot2019}, and SLEUTH trace causal paths in provenance graphs using expert-defined attack patterns. SPARSE~\cite{sparse2024} constructs suspicious semantic graphs using semantic tags and threat intelligence. Recent learning-based methods also move toward path- or sequence-level detection. SLOT~\cite{slot2025} performs anomaly detection and then constructs attack chains using label-propagation clustering. CAGE~\cite{cage2025} assigns dynamic causal weights to provenance edges with Q-learning and extracts weighted attack subgraphs. EdgeTrace~\cite{edgetrace2025} performs semantic clustering and causal path inference after edge-level anomaly scoring. Another related branch feeds provenance-derived sequences into Transformer or similar architectures, such as EagleEye~\cite{eagleeye2024} with fixed-window event sequences, Sentient~\cite{sentient2026} with graph Transformer and intent analysis, GET-AID~\cite{getaid2025} with temporal-subgraph attention, and PanThreat~\cite{panthreat2025} and LogShield~\cite{logshield2024} with log/provenance sequence modeling.

This line is closer to the needs of attack investigation because it tries to answer how an attack unfolds. The key difference lies in how the chain or sequence is produced. Some methods depend on handcrafted attack templates or threat intelligence, which limits coverage of unknown APTs. Some reconstruct paths only after node- or edge-level alerts, so chain quality is bounded by upstream recall. Others define sequences by time windows, random walks, or temporal subgraphs, which need not guarantee real dependency between adjacent events. In other words, these methods recognize the importance of process-level modeling, but the process itself is often determined by templates, segmentation rules, or post-processing rather than by an explicit and computable causal-chain quality criterion before detection.

\paragraph{Relation to this work.}

Our work belongs to the attack-process-driven line of detection and investigation, aiming to move APT alerts from local anomalies to interpretable behavioral processes. Unlike methods that output only anomalous windows, entities, or edges, we define the final detection object as a causal event chain. Unlike methods that rely on handcrafted templates or reconstruct paths only after alerts, we extract structurally credible causal chains before detection and then decide whether each chain deviates from benign behavior. This direction raises three core challenges: avoiding irrelevant dependencies introduced by graph reachability, distinguishing structurally valid benign chains from attack chains, and evaluating whether a chain-level output recovers the attack narrative. The key feature of our design is the separation between causal-chain extraction and chain-level anomaly detection: the former ensures that the detection object has an interpretable dependency structure, while the latter uses a self-supervised Transformer to learn normal chain-level behavior and detect semantic anomalies. The detailed method is presented in \S\ref{sec:method}.
